Safe defaults for Claude Code

Stop your coding agent before it does something dangerous.

Certior Guard checks every Claude Code action before it runs — blocks secrets and dangerous commands, asks before deploys and migrations, and logs every decision. Zero dependencies. Two-minute install.

$ npx certior-guard init
Get started → View on npm

“Don’t” is not a security boundary.

Coding agents now have a shell, your files, and your cloud credentials. Prompt-injected by a poisoned issue or just careless, an agent will read your .env, run curl | bash, drop a table, or deploy to prod. It’s not misalignment — nothing bounds what the agent is allowed to do. Certior is that boundary.

Block. Ask. Prove.

One PreToolUse hook. Every Bash / Edit / Read / WebFetch / MCP call is checked first — with a reason and a local receipt. Choose a profile (personal / team / production / regulated) and a mode (observe / ask / enforce).

Block

Dangerous actions, stopped

Reading secrets, curl | bash, rm -rf, disk wipes, data exfiltration — refused before they run. Evasion-resistant: it parses commands, not strings.

Ask

Risky actions, held

Deploys (terraform/kubectl/vercel), migrations, git push, dependency installs, edits to auth/billing/CI — paused for your approval.

Prove

Every decision, logged

A tamper-evident, hash-chained audit trail you can verify — proof of what your agent was allowed to do, not just a log. Local, grep-able, no cloud.
